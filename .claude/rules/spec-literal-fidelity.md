# spec.md의 사용자 노출 문자열은 원문 그대로 옮긴다

## 규칙

`spec.md`의 시나리오·성공 기준에 있는 사용자에게 보이는 정확한 문자열(버튼 라벨, 배지 텍스트, 에러 메시지 등)을 `plan.md`나 구현 코드로 옮길 때 **요약·압축·재타이핑하지 않고 원문을 그대로 복사**한다.

## 이유

`/draft-plan`이 spec.md의 문구를 plan.md 데이터 모델에 옮기며 공백을 압축하는 식으로 살짝 바꾸면(예: "적극 추천" → "적극추천"), `/execute-plan`은 plan.md만 보고 구현·테스트를 작성하기 때문에 그 차이를 그대로 물려받는다. 그 결과 유닛 테스트·E2E 테스트가 전부 통과하는데도 실제 화면 문구가 spec.md 원문과 달라진 채로 여러 커밋을 통과할 수 있다 (game-explorer Task 4, code review에서 발견).

## 적용 방법

- plan.md 작성 시 spec.md의 인용 문자열은 복사-붙여넣기하고, 손으로 다시 타이핑하지 않는다.
- 구현 코드에서 해당 문자열이 나오는 지점(타입 리터럴, JSX 텍스트, 테스트 assertion)은 spec.md 원문과 diff 가능한 수준으로 동일해야 한다.
- code-reviewer 또는 human review 단계에서 spec.md의 인용된 문자열과 실제 화면 텍스트를 대조하는 점검을 포함한다.
