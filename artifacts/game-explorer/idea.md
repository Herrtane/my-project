# 게임 탐색기 (Game Explorer)

## Problem Statement
HMW — 정적인 인기 순위 스냅샷이 아니라, 장르별로 지금 뜨고 있는 게임을 탐색하고 "살까 말까"까지 직접 판단해주는 흐름을 만들 수 있을까?

## Recommended Direction
장르/태그 필터 → 인기 순위 리스트 → 게임 클릭 시 **구매 추천 배지 + 가격·평점·할인 상세**를 보여주는 모달로 이어지는 단일 흐름의 "게임 탐색기"를 만든다. Steam 공식 API는 태그 검색을 JSON으로 제공하지 않으므로, **SteamSpy API**(무료·키 불필요, 장르별 게임 리스트 + 추정 플레이어 수 + 가격 + 리뷰 긍정률 제공)를 주 데이터 소스로 채택한다.

구매 추천은 **할인율 + 리뷰 긍정률(평점)** 을 결합한 투명한 규칙 기반 스코어로 계산해 **3단계 배지("적극 추천" / "신중히 고려" / "비추천")** 로 보여준다. AI 판단이 아니라 명시적 공식이라 설명 가능하고 구현도 빠르다. 다른 사람도 쓸 수 있는 공개형 사이트를 가정하므로 반응형·로딩/에러 상태도 신경 쓴다.

차별점은 SteamDB/SteamCharts류의 "순위 나열"에서, "장르로 좁혀서 지금 뜨는 것을 보고 배지 하나로 구매 판단까지" 이어지는 한 흐름에 있다.

## Key Assumptions to Validate
- [ ] SteamSpy genre 엔드포인트가 안정적으로 응답하고, 필터 UI에 쓸 만한 장르/태그 데이터를 제공한다 — API 직접 호출해서 응답 스키마 확인
- [ ] SteamSpy가 제공하는 할인율·리뷰 긍정률 데이터가 추천 배지 공식에 쓸 만큼 신뢰할 수 있다 — 상세 모달 구현 전 응답 샘플 확인
- [ ] 3~5시간 내 spec→wireframe→plan→build 전체 사이클을 이 스코프로 완료할 수 있다 — 첫 API 연동 후 남은 시간 재평가

## MVP Scope
**포함**: 장르/태그 필터 UI, 필터링된 인기 게임 리스트(순위), 게임 클릭 시 구매 추천 배지(3단계) + 가격·평점·할인 상세 모달, 다크모드.
**제외(스트레치)**: 랜덤 추천 버튼 — 시간이 남으면 추가.

## Not Doing (and Why)
- 순위 변동(상승/하락) 표시 — 두 시점 데이터를 비교·저장해야 해서 3~5시간 스코프에 안 맞음
- 실시간 동시접속자 수 — SteamSpy는 추정치 데이터라 완전한 실시간성은 포기
- 로그인/개인화/즐겨찾기 — 계정 관리는 이번 실습 범위 밖
- 외계행성/우주 관련 기능 — 이번 라운드는 게임으로 확정, 우주는 다음 기회에

## Open Questions
- SteamSpy API의 실제 응답 스키마(필드명, 장르 목록, 리뷰 긍정률 필드)를 확인해야 필터 UI와 추천 공식 설계가 가능함
- "인기 순위"의 기준을 SteamSpy의 어떤 필드(예: owners 추정치, score)로 정할지 결정 필요
- 추천 배지의 정확한 임계값(예: 할인율 몇 % 이상 + 리뷰율 몇 % 이상 = "적극 추천")은 spec 단계에서 구체화 필요
